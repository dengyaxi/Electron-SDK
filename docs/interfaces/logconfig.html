Interface LogConfig

SDK 日志文件的配置。

since

v3.3.1

Hierarchy

  • LogConfig

Index

Properties

  • filePath
  • fileSize
  • level

Properties

filePath

filePath: string

日志文件的完整路径。

请确保你指定的目录存在而且可写。你可通过该参数修改日志文件名。

fileSize

fileSize: number

单个日志文件的大小，单位为 KB。默认值为 1024 KB。如果你将 fileSize 设为 1024 KB，SDK 会最多输出总计 5 MB 的日志文件。 如果你将 fileSize 设为小于 1024 KB，设置不生效，单个日志文件最大仍为 1024 KB。

level

level: number

SDK 的日志输出等级:

  • 0x0000: 不输出任何日志。
  • 0x0001: (默认) 输出 FATAL、ERROR、WARN、INFO 级别的日志。我们推荐你将日志级别设为该等级。
  • 0x0002: 仅输出 FATAL、ERROR、WARN 级别的日志。
  • 0x0004: 仅输出 FATAL、ERROR 级别的日志。
  • 0x0008: 仅输出 FATAL 级别的日志。
